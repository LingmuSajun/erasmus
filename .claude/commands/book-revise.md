# 修正

レビュー指摘事項に基づいて原稿を修正してください。

引数なしの場合: 未修正の章のうち、指摘が最も重い章から修正する。
引数ありの場合: `$ARGUMENTS` で指定された章を修正する。

- `1` 〜 `N` → 該当する章を修正
- `intro` / `end` → はじめに / おわりに を修正
- `all` → 指摘のあるすべての章を順番に修正する

## 前提条件

- `output/reviews/` に最新ラウンドのレビューファイルが存在すること

## 手順

1. `progress.yml` を読み、現在のレビューラウンドと修正状況を確認する
2. 対象章のレビューファイル（`review-round{N}-ch{章番号}.md`）を読む
3. 全体整合性チェック（`output/reviews/review-round{N}-overall.md`）があれば、対象章に関する指摘も確認する
4. `book-theme.md` を読み、トーン・口調を再確認する
5. `book-config.yml` を読み、文字数制約を再確認する
6. 対象章の原稿（`output/drafts/`）を読み込む
7. 指摘事項を重要度順（高→中→低）に対応する
8. 修正後の原稿をファイルに上書き保存する
9. `progress.yml` を更新する

## 修正ルール

- 重要度「高」の指摘: 必ず対応する
- 重要度「中」の指摘: 可能な限り対応する
- 重要度「低」の指摘: 文脈を見て判断する
- 修正後も文字数が `book-config.yml` の設定範囲（±10%）に収まるようにする
- 修正により新たな矛盾が生じないよう、前後の章の冒頭・末尾を確認する
- 元の文章の良い部分は保持し、指摘箇所のみ改善する

## 1章あたりの修正が大きい場合

修正量が章全体の30%を超える場合は、セクション単位で修正する:

1. 修正が必要なセクションを特定する
2. セクション単位で修正し、ファイルに書き戻す
3. 修正後のセクション間の接続を確認する

## 完了後

以下を表示する:

- 対応した指摘事項の数（高/中/低それぞれ）
- 対応しなかった指摘事項とその理由（あれば）
- 修正後の文字数
- 全体の修正進捗（例: 5/8章 修正完了）
- 次のステップの案内
  - 未修正の章がある → `/book-revise` で次の章へ
  - 全章修正完了 → `/book-re-review` へ
