# 執筆

指定された章の本文を執筆してください。

引数なしの場合: 未執筆の章を1つ選んで執筆する。
引数ありの場合: `$ARGUMENTS` で指定された章を執筆する。

- `intro` または `はじめに` → はじめに
- `1` 〜 `N` → 該当する章
- `end` または `おわりに` → おわりに
- `all` → 未執筆の全章を順番に執筆する

## 前提条件

- `output/outline.md` が存在すること（なければ `/book-outline` を案内する）

## 手順

1. `progress.yml` を読み、現在の執筆状況を確認する
2. `book-config.yml` を読み、文字数・セクション分割設定を確認する
3. `book-theme.md` を読み、トーン・口調・キーワードを確認する
4. `output/outline.md` を読み、対象章の構成を把握する
5. 既に執筆済みの前後の章がある場合、冒頭・末尾を確認して接続を意識する
6. セクション単位で分割して執筆する（後述）
7. すべてのセクションを結合して1つの章ファイルとして保存する
8. `progress.yml` を更新する

## セクション分割執筆

1章の文字数が `sections.max_chars_per_output`（デフォルト4,000字）を超える場合、
章をセクション単位に分けて段階的に執筆する。

### セクション分割の手順

1. まず章全体のセクション構成（見出し・各セクションの概要）を決定する
2. セクション1を執筆し、ファイルに書き込む
3. セクション2を執筆し、ファイルに追記する
4. 以降、すべてのセクションが完了するまで繰り返す
5. 最後に章全体を通読し、セクション間の接続を微調整する

### 各セクション執筆時の注意

- 前のセクションの末尾を読み、文脈が自然につながるようにする
- `progress.yml` の `sections_completed` をセクション完了ごとに更新する
- セッションが中断されても、途中から再開できるようにする

## 各章の執筆ルール

- `book-config.yml` で指定された文字数を目安にする（±10%）
- 章の冒頭: 読者を引き込む導入（エピソード、問いかけ、意外な事実など）
- 章の本文: アウトラインで指定されたトピックをすべて網羅する
- 章の末尾: その章の要点を3〜5個の箇条書きでまとめる
- 前後の章との接続: 前章の内容を軽く振り返りつつ、本章のテーマへ自然につなげる
- 具体例・データ・事例を積極的に使う（ただし捏造しない）
- `book-theme.md` の「避けるべきこと」に該当する表現を使わない

## ファイル命名規則

- `00-はじめに.md`
- `01-第1章.md`, `02-第2章.md`, ...
- 最終: `{章数+1のゼロ埋め2桁}-おわりに.md`

## 各ファイルのフォーマット

```markdown
# 《章タイトル》

## セクション1タイトル

（本文）

## セクション2タイトル

（本文）

...

---

## この章のポイント

- ポイント1
- ポイント2
- ポイント3
```

## 完了後

以下を表示する:

- 執筆した章のファイル名と文字数
- セクション数
- 全体の進捗状況（例: 3/8章 完了）
- 次のステップの案内
  - 未執筆の章がある → `/book-write` で次の章へ
  - 全章完了 → `/book-review` へ
