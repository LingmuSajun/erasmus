# レビュー

執筆済みの原稿をレビューしてください。

引数なしの場合: 未レビューの章を1つ選んでレビューする。
引数ありの場合: `$ARGUMENTS` で指定された対象をレビューする。

- `1` 〜 `N` → 該当する章を個別レビュー
- `intro` / `end` → はじめに / おわりに を個別レビュー
- `all` → 全章を順番にレビューし、最後に全体整合性チェックを行う
- `overall` → 全体整合性チェックのみ実行する

## 前提条件

- `output/drafts/` に原稿ファイルが存在すること
- すべての章が `draft_completed` であること（`progress.yml` で確認）

## レビューは2段階で行う

### Stage 1: 章単位レビュー

各章を個別にレビューする。1章ずつコンテキストに読み込むため、長い原稿でも品質を維持できる。

#### Stage 1 の手順

1. `review-criteria.md` を読み、レビュー基準を確認する
2. `book-theme.md` を読み、テーマ・トーンとの整合性基準を確認する
3. `output/outline.md` を読み、対象章のアウトラインを確認する
4. 対象章の原稿を読み込む
5. 以下の観点でレビューする:
   - `review-criteria.md` の基準への適合
   - 文字数が `book-config.yml` の設定範囲内か
   - アウトラインで指定されたトピックをすべてカバーしているか
   - 導入部は読者の興味を引けているか
   - 具体例や事例は十分か
   - 章末まとめは本文の内容を正しく反映しているか
   - トーンと口調は `book-theme.md` に沿っているか
6. 結果を `output/reviews/review-round{N}-ch{章番号}.md` に保存する
7. `progress.yml` を更新する

### Stage 2: 全体整合性チェック

全章のレビューが完了した後に実行する。全文を読み込む代わりに、
各章の要約（章末まとめ + 冒頭200字 + 末尾200字）を使って全体の整合性を確認する。

#### Stage 2 の手順

1. 各章から以下を抽出する:
   - 章タイトル
   - 冒頭200字
   - 末尾200字
   - 「この章のポイント」セクション
2. これらを統合して全体像を把握する
3. 以下の観点でチェックする:
   - 章間の論理的な流れは自然か
   - 同じ内容の不必要な繰り返しはないか
   - トーンと口調は全体を通して一貫しているか
   - キーワードが適切に使われているか
   - 「はじめに」で提示した問いに「おわりに」で答えているか
   - 章の順序は適切か（入れ替えた方が良い箇所はないか）
4. 結果を `output/reviews/review-round{N}-overall.md` に保存する

## 章単位レビューの出力フォーマット

```markdown
# レビュー: {章ファイル名}（ラウンド {N}）

## 評価
（A: 完成レベル / B: 軽微な修正で完成 / C: 要修正）

## 良い点
- ...

## 指摘事項

### [重要度: 高] 指摘タイトル
- 該当箇所: 「...」
- 問題点: ...
- 改善案: ...

### [重要度: 中] 指摘タイトル
（同上）

### [重要度: 低] 指摘タイトル
（同上）
```

## 全体整合性チェックの出力フォーマット

```markdown
# 全体整合性チェック（ラウンド {N}）

## 全体評価
（A / B / C）

## 章間の流れ
- ...

## 重複・矛盾の検出
- ...

## 全体への指摘事項
- ...

## 総合判定
（「修正が必要」または「完成」）
```

## 完了後

- 章単位レビュー完了時: レビュー済み章数の進捗を表示し、次の章 or 全体チェックを案内
- 全体チェック完了時:
  - 「完成」→ `/book-finalize` を案内
  - 「修正が必要」→ `/book-revise` を案内
