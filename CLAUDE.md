# Erasmus — 書籍自動執筆システム

ルネサンスの知の巨人エラスムスの名を冠した、ノンフィクション書籍（教養書・解説書）を自動執筆するためのClaude Code環境です。

## プロジェクト構成

```markdown
erasmus/
├── CLAUDE.md              # このファイル（プロジェクト全体のルール）
├── book-config.yml        # 章数・文字数・セクション分割などの構造設定
├── book-theme.md          # テーマ・トーン・口調・キーワード（ユーザー編集）
├── review-criteria.md     # レビュー基準・観点（ユーザー編集）
├── progress.yml           # 進捗管理（自動更新。手動編集しない）
├── .claude/
│   ├── commands/          # ワークフローコマンド
│   ├── skills/            # 執筆スキル
│   └── agents/            # レビューエージェント
└── output/
    ├── outline.md         # アウトライン
    ├── drafts/            # 各章の原稿
    ├── reviews/           # レビュー指摘事項
    └── final/             # 最終成果物（.md + .pdf）
```

## ワークフロー

以下の順序でコマンドを実行する：

1. `/book-outline` → アウトライン作成
2. `/book-write [章番号|all]` → 1章ずつ本文執筆（章番号指定 or 全章順次実行）
3. `/book-review [章番号|all]` → 章単位レビュー + 全体整合性チェック
4. `/book-revise [章番号|all]` → レビュー指摘に基づく修正
5. `/book-re-review [章番号|all]` → 再レビュー（OKなら完成判定）
6. `/book-finalize` → 最終ファイル生成（.md + .pdf）

ユーティリティ:

- `/book-auto` → 上記すべてを順番に自動実行（途中再開対応）
- `/book-status` → 現在の進捗状況を表示

## 設計原則: 分割と再開

本システムは新書〜文庫本相当（7〜12万字）の分量に対応するため、以下の設計を採用する:

### 章単位の実行

- 執筆・レビュー・修正はすべて1章ずつ実行できる
- コンテキストウィンドウの圧迫を防ぎ、各章の品質を維持する
- `all` を指定した場合は全章を順番に処理する

### セクション分割執筆

- 1章の文字数が長い場合（デフォルト: 10,000字）、セクション単位に分割して執筆する
- `book-config.yml` の `sections.max_chars_per_output` で1回の出力上限を制御する

### 進捗ファイルによるセッション横断

- `progress.yml` に執筆・レビューの進捗を記録する
- セッションが中断しても、次回起動時に途中から再開できる
- `/book-status` でいつでも進捗を確認できる

### 2段階レビュー

- Stage 1: 各章を個別にレビュー（詳細な品質チェック）
- Stage 2: 全章の要約を使って全体整合性をチェック（論理の流れ、重複、矛盾）

## 基本ルール

- すべての執筆は `book-theme.md` のトーン・口調・キーワードに厳密に従う
- 章の構造・文字数は `book-config.yml` の設定に従う
- 各章は独立して読めるが、全体として一貫した流れを持つこと
- ファイルの読み書きは必ず `output/` ディレクトリ配下で行う
- 原稿ファイル名: `output/drafts/00-はじめに.md`, `output/drafts/01-第1章.md`, ...
- レビューファイル名: `output/reviews/review-round{N}-ch{章番号}.md`（章別）、`output/reviews/review-round{N}-overall.md`（全体）
- 最終成果物: `output/final/{タイトル}.md`, `output/final/{タイトル}.pdf`
- 進捗管理: `progress.yml`（自動更新。手動編集しない）

## 設定ファイルの読み取り順序

1. `book-config.yml` を読み、章数・文字数の制約を把握する
2. `book-theme.md` を読み、テーマ・トーン・口調・キーワードを把握する
3. `review-criteria.md` を読み、レビュー基準を把握する
4. すべての情報を統合して執筆・レビューに入る

## 文字数について

- `book-config.yml` の文字数は「目安」であり、±10%の範囲で調整してよい
- 文字数にはMarkdownの記法（#, *, - など）は含めない。本文テキストのみでカウントする
